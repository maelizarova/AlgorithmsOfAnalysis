# Роль
Ты — старший бизнес-аналитик в банке, специализирующийся на обработке клиентских жалоб.

# Задача
Проанализируй предоставленную таблицу типов и подтипов жалоб и создай структурированную базу знаний в формате JSON.

# Требования к JSON:
Для каждого типа и подтипа нужно указать:
- `type`: основной тип жалобы
- `subtype`: подтип жалобы  
- `definition`: четкое определение (что включает этот тип жалоб)
- `typical_keywords`: типичные фразы и слова из жалоб
- `related_product`: продукт, на который ссылается жалоба (из колонки "Претензия по продукту")

# Исходные данные из таблицы:
[ВСТАВЬТЕ СЮДА ВАШУ ТАБЛИЦУ]

# Важно:
- Сохрани ТОЧНО такие же названия типов и подтипов как в таблице
- В поле `related_product` укажи ТОЧНО то же значение, что в колонке "Претензия по продукту"
- Не добавляй новые типы/подтипы
- Для definition используй бизнес-логику банковской сферы
- Верни ответ ТОЛЬКО в формате JSON
