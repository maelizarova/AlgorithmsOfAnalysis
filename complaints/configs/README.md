# Как заполнять JSON-конфиг

Файл `debit_card_retail.json` — это главный переключатель проекта.
Через него задается и бизнес-контекст, и технические параметры запуска.

## Корневые поля

- `product_id` — идентификатор продукта (например, `debit_card`).
- `product_category` — категория продукта (например, `retail`).
- `input_path` — путь к входному файлу с обращениями (CSV или parquet).
- `prompts_dir` — папка с prompt-файлами для этой пары.
- `taxonomy_issues_path` — путь к справочнику проблем issues.
- `taxonomy_requests_path` — путь к справочнику запросов requested_actions.
- `text_column` — название колонки с текстом обращения во входных данных.
- `llm` — настройки LLM.
- `runtime` — настройки батчинга, ретраев и параллелизма.

## Блок `llm`

- `api_key` — ключ API.
- `base_url` — URL API-провайдера.
- `model` — имя модели.
- `temperature` — креативность генерации (обычно `0` для классификации).
- `timeout_seconds` — HTTP timeout (в коде все равно ограничивается 60 сек).
- `model_kwargs` — дополнительные параметры модели в формате JSON-объекта.

## Блок `runtime`

- `batch_size` — размер батча.
- `max_concurrency` — сколько запросов отправлять параллельно в `chain.batch`.
- `retries` — количество повторов при ошибке.
- `backoff_base_seconds` — базовая пауза для экспоненциального backoff.

## Контекст продукта

Для каждой пары продукт/категория в папке промптов (`prompts_dir`) должен лежать файл `product_context.txt`.
Он содержит текстовое описание продукта и типичных тем жалоб — это описание автоматически подставляется во все промпты через переменную `{product_context}`.
Если файла нет — промпты работают без контекста (переменная будет пустой строкой).

## Примечание

Чтобы добавить новую пару продукт/категория:
1. Создай новый JSON в `configs/`.
2. Создай папку промптов и файл `product_context.txt` с описанием продукта.
3. Создай (или скопируй) prompt-файлы и таксономии под эту пару.
4. Укажи путь к новому JSON в notebook в переменной `CONFIG_PATH`.

В текущем notebook после этапа 0b таксономии сохраняются в `results/<pair>/`.
Если нужно, их можно вручную поправить и сразу запускать следующую ячейку.
