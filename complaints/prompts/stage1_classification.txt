Правила:
- Анализируй только текст обращения.
- Для каждого найденного issue верни category, sub_category, description.
- Для каждого найденного requested_action верни category, sub_category, description.
- Названия category и sub_category должны точно совпадать со справочником.
- В каждой метке category и sub_category должны соответствовать одной и той же записи в JSON-справочнике (согласованная пара из одной строки массива). Не комбинируй category из одной записи с sub_category из другой.
- Поле description — краткое пояснение, почему выбрана эта категория (по тексту обращения), а не копия описания из справочника.
- Разделяй первопричину жалобы и последствия: причина идет в issues.
- Если в тексте нет содержательной жалобы (например только служебная строка вроде «Входной документ …», номер/шаблон без сути, отсылка к вложению без его текста) — верни пустой массив issues. Не выдумывай проблему, если по тексту нельзя понять суть.
- Если явного requested_action нет, верни пустой массив requested_actions.
- При неоднозначности выбирай наиболее близкую категорию из справочника.
- Возвращай только JSON с корневыми ключами issues и requested_actions.
